# Handoff: Line Test Manager — 공정 Line Test 관리 앱

## Overview
공정 Line Test 시 차수(Run)별 공정조건, 물성평가 결과, 특이점 사진을 스마트폰으로 빠르게 입력하고, 차수 간 데이터를 비교·트렌드 분석하는 모바일 앱입니다.

혼자 사용하는 단일 사용자 앱으로, 현장에서 실시간 입력이 핵심입니다.

---

## About the Design Files

`Line Test Wireframe.html`은 **로우피델리티(lo-fi) 와이어프레임 레퍼런스**입니다.
- React + Babel 인라인으로 작성된 인터랙티브 와이어프레임이며, 프로덕션 코드가 아닙니다.
- 개발자는 이 파일을 **구조·흐름·데이터 구조의 참고용**으로만 사용하고, 실제 구현은 팀의 기존 코드베이스(React Native, Flutter, SwiftUI 등)에서 해당 환경의 패턴과 라이브러리로 재구현해야 합니다.
- 스타일(색상, 폰트, 간격)은 기존 디자인 시스템을 우선 적용하세요.

## 앱 구조 (Information Architecture)

```
Line Test Manager
├── 홈 (테스트 목록)
│   ├── 진행중 테스트
│   └── 완료된 테스트
│
├── 테스트 상세 (차수 목록)
│   ├── 진행 현황 스트립 (Run 1~N 도트)
│   ├── 차수(Run) 목록
│   │   ├── 완료 Run
│   │   ├── 진행중 Run (하이라이트)
│   │   └── 대기 Run
│   ├── [비교 보기] 버튼
│   └── [트렌드] 버튼
│
├── 차수 입력 폼 (핵심 화면)
│   ├── 탭: 공정조건 / 물성평가 / 사진·특이점
│   ├── Test Leg 선택 (A/B/C)
│   └── 이전 Run 복사 버튼
│
└── 비교·트렌드
    ├── 탭: 차수 비교 / 트렌드
    ├── 비교 축: Run별 / Leg별
    └── 항목 필터: 전체 / 변경 항목만
```

## 핵심 개념 정의

| 개념 | 설명 |
|---|---|
| **Line Test** | 하나의 공정 테스트 프로젝트 (예: PE-LLD #23-A) |
| **Run (차수)** | Line Test 내 개별 실험 회차 (Run 1, Run 2…) |
| **Test Leg** | 하나의 Run 내에서 다른 제품 등급이나 조건을 동시에 테스트하는 병렬 단위 (Leg A, Leg B, Leg C) |
| **공정조건** | 온도, RPM, 압력 등 운전 파라미터 |
| **물성평가** | MI, 밀도, 외관 등 제품 분석 결과 |
| **Target** | 사전에 설정된 목표값 (읽기전용) |
| **공차 (±)** | 허용 편차 범위 |
| **실측값** | 현장에서 측정·입력하는 실제 값 |

## Screen 1: 홈 — 테스트 목록

### Layout
- 상단: 검색 바
- 섹션 구분: "진행중" / "완료됨"
- 리스트 아이템: 테스트명, 제품명, 차수 수, 최근 날짜, chevron(›)
- 진행중 아이템: 노란 배경 강조
- FAB (+): 화면 우하단 고정, 신규 테스트 생성

### Interactions
- 리스트 탭 → Screen 2 (차수 목록)
- FAB 탭 → 신규 테스트 생성 모달/화면

---

## Screen 2: 테스트 상세 — 차수 목록

### Layout
- 상단 바: 테스트명 + 편집 버튼
- 진행 현황 스트립:
  - 차수 번호 도트 (완료=검정, 진행중=노랑, 대기=회색 테두리)
  - 도트 사이 연결선 (완료 구간=검정, 미완료=회색)
- 상태 배지: `조건변경` (노란 칩), `진행중` (빨간 칩)
- 하단 고정 버튼: "+ 새 차수 입력 시작"

### State per Run
```typescript
interface Run {
  id: number;
  label: string;        // "Run 1"
  note: string;         // "T3 온도 +5°C" 등 변경 요약
  status: 'done' | 'active' | 'pending';
  hasConditionChange: boolean;
  startTime: string;
}
```

## Screen 3: 차수 입력 폼 (핵심 화면)

### Layout 개요
```
[상단 바: 테스트명 · Run N | 임시저장]
[이전 Run 변경 항목 칩 스트립]
[탭: ⚙️공정조건 | 🧪물성평가 | 📷사진·특이점]
[Test Leg 선택 탭: Leg A | Leg B | Leg C | + 추가]
[컬럼 헤더: 항목 | Target | 공차 | 실측값(선택된 Leg)]
[폼 행 목록]
[하단: ← 이전 Run 복사 | 저장 →]
```

### 공정조건 탭 — 폼 행 구조

각 행은 4열 구성:
```
[변경도트] [항목명] [Target(읽기전용)] [공차(±)] [실측값(입력)]
```

| 열 | 내용 | 너비 | 스타일 |
|---|---|---|---|
| 변경 도트 | 이전 Run 대비 변경된 항목 표시 | 7px | 빨간 원 |
| 항목명 | T1 온도, RPM 등 | ~76px | 회색 텍스트 |
| Target | 목표값 (읽기전용) | ~52px | 회색 배경, 연한 테두리 |
| 공차 | ±N (편집 가능) | ~44px | 점선 테두리, 노란 텍스트 |
| 실측값 | 현장 입력값 | flex:1 | 흰 배경, 검정 테두리 |

**조건부 스타일:**
- `changed=true`: 행 배경 노란색(`#fff8e0`), Target 테두리 주황
- Target 이탈 (실측 - Target > 공차): 실측값 칸 빨간 테두리 + `!` 표시
- 미입력: 실측값 칸에 `—` 표시, 회색 테두리

**공정조건 섹션 그룹:**
1. 공정 온도: T1~T4 온도, Die 온도
2. 운전 조건: RPM, 압력, 투입량, 진공도, 냉각수 온도

**물성평가 탭 — 같은 4열 구조:**
1. 분석값: MI, 밀도, 외관, 인장강도(MD), 헤이즈

### Test Leg 동작
- Leg 선택 시 해당 Leg의 실측값만 표시·수정
- 공정조건/물성 탭 모두 동일한 Leg 선택 공유
- Leg 추가 가능 ("+추가" 버튼)

### 이전 Run 변경 항목 스트립
- 이전 Run과 비교해 달라진 조건을 노란 칩으로 나열
- 예: `T3 온도 +5°C`, `RPM -20`

### 사진·특이점 탭
- 사진 썸네일 가로 스크롤 (카메라 촬영 + 갤러리 선택)
- 특이점 메모 텍스트 영역
- 태그 선택: 외관불량, 버블, 압력이상, 온도이상, 이물질, 기타

### Data Model
```typescript
interface RunData {
  runId: number;
  testId: string;
  timestamp: string;
  conditions: {
    [itemId: string]: {
      target: number | string;
      tolerance: string;       // "±2", "≥", "≤" 등
      actuals: {
        [legId: string]: number | string;
      };
    };
  };
  properties: {
    [itemId: string]: {
      target: number | string;
      tolerance: string;
      actuals: {
        [legId: string]: number | string;
      };
    };
  };
  photos: Photo[];
  memo: string;
  tags: string[];
}

interface ConditionItem {
  id: string;
  label: string;
  unit: string;
  group: 'temperature' | 'operation';
  isTemplate: boolean;  // 템플릿 기반 vs 자유 추가
}
```

## Screen 4: 비교·트렌드

### 차수 비교 탭

**비교 축 전환 (토글):**
- **Run별 비교**: Leg 고정 선택 → 컬럼 = R1, R2, R3, R4
- **Leg별 비교**: Run 고정 선택 → 컬럼 = Leg A, Leg B, Leg C

**필터:** 전체 항목 / 변경 항목만

**테이블 계층 구조:**
```
⚙️ 공정조건 (헤더 행)
  └ 공정 온도 (서브헤더)
      ├ T1 온도 | 목표:185 ±2 | R1 | R2 | R3 | R4
      ...
  └ 운전 조건 (서브헤더)
      ...
🧪 물성평가 (헤더 행)
  └ 분석값 (서브헤더)
      ...
```

**셀 조건부 스타일:**
- 이전 컬럼 대비 값이 바뀐 셀: 노란 배경, 굵은 주황 텍스트
- `버블` 등 불량 값: 빨간 배경, 빨간 텍스트
- 미입력: `—` 표시

### 트렌드 탭

**차트 구성:**
- X축: Run 번호 (R1~R4)
- Y축: 선택 항목의 실측값
- Leg별 3개 선: Leg A (실선), Leg B (긴 점선), Leg C (짧은 점선)
- 목표값 기준선: 노란 점선
- 이탈 포인트: 빨간 도트 + 말풍선 어노테이션

**항목 선택:** 칩 형태 (MI, 밀도, RPM, T3온도 등)

## 핵심 UX 원칙

1. **빠른 입력 최우선**: 현장에서 한 손으로 입력 가능해야 함. 숫자 항목은 숫자 키패드 자동 활성화.
2. **이전 Run 복사**: 신규 차수 시작 시 이전 값을 그대로 복사해 변경 항목만 수정.
3. **변경 항목 시각화**: 조건이 바뀐 항목은 항상 강조 표시 (도트 + 배경색).
4. **공차 이탈 즉시 피드백**: 실측값이 Target ± 공차 범위를 벗어나면 입력 즉시 경고.
5. **임시저장**: 폼 이탈 시 자동 임시저장, 재진입 시 복원.
6. **Leg 전환**: 공정조건/물성 탭에서 Leg 선택이 항상 상단에 고정 노출.

## 템플릿 시스템

- 공정조건·물성 항목 세트를 "템플릿"으로 저장·재사용 가능
- 신규 테스트 생성 시 템플릿 선택
- 항목 추가 시 "이 테스트에만 추가" vs "템플릿에도 저장" 선택
